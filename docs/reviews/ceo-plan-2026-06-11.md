## Vision

### 10x Check
The 10x version is not a prettier dashboard — it is the **action queue that closes the loop**: offices are pushed (digest/badge) the 2-3 patients that need THEIR action with exact next steps, data flows in automatically (Sheets API → eventually direct workflow feed), and Insight's admin sees account health and SLA risk computed from documentation-complete timestamps. The extension becomes one head of a channel-agnostic status API. Both outside voices (Claude subagent + Codex GPT-5.5) independently converged on this reframing: visibility → resolution, pull → push.

### SOP Grounding
SOP-SPI-001 v2.0 (3 documents: marketing one-pager + 2 internal layouts, substantively identical) defines the 4-phase workflow (Enrollment/Submission same-day → Intake/PA 24-48h* → Telehealth/Approval 24h → Fulfillment/Delivery 24-48h; *complex PA 2-3 weeks measured from complete documentation). The marketing variant (controls stripped) is the provider-facing vocabulary source — the extension is effectively the interactive marketing SOP. The internal variants confirm: monthly HIPAA audits exist; timestamps are audit-trail mandatory; escalation if patient unresponsive >24h; KPIs = eRx-to-ship, no-show rate, incomplete script %, PA denial rate.
